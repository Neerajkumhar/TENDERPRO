Critical Audit Findings & Issues

RESOLVED: Secured API Routes with Authorization Middleware SECURED

We implemented a JWT-based token verification middleware (authMiddleware.js) and applied it globally to all private REST routes in server/index.js. We also updated the frontends (client and admin) using a global fetch interceptor in their entry points (main.jsx) to automatically attach the bearer token to all outgoing requests. All business routes now require valid credentials.

Resolved: Database Initialization Sync Timeout RESOLVED

The server previously crashed with a Database initialization failed: Database sync timed out error on boot. This was due to a strict 15-second timeout on sequelize.sync({ alter: true }). Since the DB is on remote AWS RDS, comparing 20+ models sequentially took ~4 minutes. We resolved this by increasing the boot timeout limit to 5 minutes for development servers.

Resolved: Missing Tender Manager Association RESOLVED

Seed scripts originally stored internal team assignments as a standard array, while the client controllers and frontend views expected a JSON object structure containing managerId, reviewerId, and approverId. We corrected the seeding data structure and applied updates across the database. Tenders now show proper manager/reviewer details.

Actionable Roadmap

  • Migrate to SQL Migration Files: Instead of checking the database schema programmatically on every single boot using sync({ alter: true }), write specific migrations via sequelize-cli. This removes the 4-minute boot lag and secures schema stability.
  • Add Database Indexes: Place indexes on foreign keys (such as clientId on the Tenders table) to maximize search performance as data grows.
  • Automated Testing: Set up a unit testing framework (like Jest or Mocha) to run our validation queries on local commits and pull requests automatically.
